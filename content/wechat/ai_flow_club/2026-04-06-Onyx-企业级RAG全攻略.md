其底层逻辑本质是 **“Connected AI Workspace (连通式 AI 空间)”**：
1.  **它不仅仅有模型**：它能跟你的飞书、钉钉、GitLab、Google Drive 甚至你私有的 SQL 数据库直接“握手”。
2.  **它不会“断篇儿”**：它自带高效的向量搜索组件。不管你是有 1 万份 PDF 还是 10 万行代码，它都能在 0.5 秒内精准锁定。
3.  **它不仅能问答**：它能开启 **Deep Research** (深度研究) 模式。你可以问它：“对比一下我们去年和前年的服务器成本，并找出异常增长的月份。这叫 **Onyx Intelligence**。

## 🛠️ 详细详细的操作步骤：如何低成本搭建你的“私有智库”？

别被“企业级”三个字吓跑，Onyx 对个人和中小创业团队非常友好。

### 第一步：架设“大脑基座” (部署)
我们推荐使用 Docker 方式，一键搞定复杂的数据库和环境。
```bash
git clone https://github.com/onyx-dot-app/onyx.git
cd onyx/deployment/docker-compose
docker-compose up -d
```
打开你的服务器地址 `localhost:3000`，一个深邃的蓝色界面就出来了。

### 第二步：插上电源（配置连接器）
去后台的 `Connectors` 模块。
1. **飞书 (Lark/Feishu)**：点击添加，填入你的 App ID。勾选你想让 AI 学习的文档库。
2. **GitHub**：填入你的个人访问令牌，选定几个核心项目仓库。
**这时候，你会看到加载条在飞速旋转，那是 AI 正在把你过去的“废弃文档”嚼碎、消化、存入它的赛博大脑。**

### 第三步：正式开机（对话与研究）
不要只是打个招呼。试着输入：“总结下我们所有的 API 鉴权流程，并写一份针对新人的新手引导。”
你会发现，它不仅流利地写出了内容，**每一句后面都带有一个小括号，标注了信息的出处。这叫可信 AI。**

**价值**：这不仅是提速，这是**“信息平权”**。

## 🔥 核心逻辑本质：爆款提示词 (The Onyx Enterprise Prompt)

即便你在用 Dify 之类的工具，也可以借鉴 Onyx 的这套“工业级 RAG 逻辑”给你的 AI 下指令：

> **[身份设定]**：你是一位拥有 10 年经验的大型企业“知识审计员”。
> 
> **[核心职责 - Onyx Strategy]**：
> 1. **强制源对齐**：任何回答必须标注引用源文件的 [文件名 + 段落摘要]。禁止基于训练数据的常识进行猜测。
> 2. **冲突识别**：如果两份文档对同一个规则的描述不一致，必须在回答中明确标记 [Conflict Warning: 文件 A 与 文件 B 冲突]。
> 3. **意图路由**：在回答前，请先告诉我你打算在哪个“分库”里查找（技术库 vs 财务库 vs 政策库）。
> 
> **[输出调性]**：严谨、多维度、带引用链接。
