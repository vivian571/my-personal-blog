# 别再只刷 Prompt 了！CLI-Anything：让万物皆可被 AI 自动化调用的终极方案

### 01. 2026：Agent 觉醒之年的“残障”陷阱

站在 2026 年 4 月的时间节点回望，我们不得不承认：尽管大语言模型（LLM）的逻辑推理能力已经逼近人类极限，但在“干脏活、累活”这件事上，Agent 依然表现得像个偏科的优等生。

你问它如何优化代码，它能给你滔滔不绝讲出八种设计模式；但你让它：

- “把这段录音里的气声去掉，并合成一段带有悲凉感的背景音。”
- “在 Blender 里按这个草图渲染出一个光追级别的 3D 模型。”
- “把这份 500 页的 Markdown 稿件排版成符合 IEEE 标准的样刊 PDF。”

它往往会陷入长久的沉默，或者丢给你一堆逻辑混乱、满是 Bug 的 Python 爬虫脚本。

**为什么？因为 AI 懂推理，但它的“手脚不协调”。**

现有的 GUI 软件架构是为了让人类用手指点击而设计的，不是为了让 Agent 流畅调用。这就是我今天要聊的——**CLI-Anything**。

---

### 02. “降维打击”：从 GUI 到 Agent-Native 的暴力美学

![Software Layers](https://images.unsplash.com/photo-1551033406-611cf9a28f67?ixlib=rb-1.2.1&auto=format&fit=crop&w=1200&q=80)

作为一名写了 20 年代码的老兵，我一直推崇“极简主义”。**CLI (命令行)** 是计算机历史上最伟大、也最稳健的协议。

由香港大学数据科学学院（HKUDS）提出的 **CLI-Anything** 项目，其核心哲学只有一句话：**“把万物变成 Agent 喜欢的样子。”**

它不再试图通过脆弱的 UI 自动化（像素点击、坐标定位）去模拟人类，而是通过一套名为 **HARNESS** 的 7 阶段标准流程，直接把复杂的专业软件（GIMP, Blender, LibreOffice, Kdenlive...）暴力拆解，再重构成一套极其轻量、结构化的 CLI 接口。

#### 这意味着什么？

你的 Agent 不再需要理解什么是“右键点击 -> 滤镜 -> 模糊”，它只需要执行：

```bash
cli-anything-gimp --json image blur --radius 15 --input raw.png
```

**JSON 输出、状态回滚（Undo/Redo）、确定性执行。** 这才是 2026 年程序员该有的开发姿势。

### 03. 实战复盘：从 GIMP 涂鸦到 3D 建模的“秒级”跨越

最近我在『AI流习社』的内部实验室里部署了这套环境。最让我震撼的不是它的速度，而是它的 **“自我进化”能力**。

以前，给视频加字幕需要我手动调整 Timeline。现在，我通过 CLI-Anything 挂载了 `VideoCaptioner`：

1.  **Agent 自动识别**：音频流被切片，Whisper-v4 毫秒级生成文本。
2.  **CLI 精确压制**：Agent 调用命令调整字体、颜色、呼吸感，不再有坐标漂移。

甚至当你遇到一个从来没有 API 的老旧 ERP 软件，只要你把源码（或者哪怕是反编译后的关键逻辑）丢给它，执行 `/cli-anything` 指令，它会经历这一场蜕变：

- **Phase 1-3 (分析与设计)**：Agent 把软件内部的所有函数映射成 CLI 命令组。
- **Phase 4-6 (自动化测试)**：它会自己写单元测试，验证命令是否输出了正确的 JSON。
- **Phase 7 (发布)**：一个崭新的 `pip install` 包就此诞生。

**这种从“寻找工具”到“制造工具”的跨越，才是 AI Agent 真正的降临时刻。**
