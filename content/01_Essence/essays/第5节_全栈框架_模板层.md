# 第5节：全栈框架_模板层

## 来源

模板层是Django框架中负责生成用户界面的重要组件。在Django的MTV（Model-Template-View）架构中，模板负责定义如何向用户展示数据。Django的模板系统设计简洁而强大，提供了变量插值、控制流、继承等功能，同时保持了HTML代码的简洁性。模板系统的设计理念是将表现逻辑与业务逻辑分离，使前端开发者和后端开发者能够更好地协作。

## 定义

### 模板层的概念

模板层是Web应用程序中负责生成HTML（或其他格式）响应的组件。在Django中，模板层由以下部分组成：

1. **模板文件**：包含HTML代码和特殊语法的文件，通常以`.html`为扩展名。
2. **模板语言**：Django模板语言（DTL），包含变量、标签、过滤器等。
3. **上下文**：传递给模板的变量集合。
4. **模板引擎**：负责解析模板文件并渲染最终输出的组件。

### 模板语法

#### 变量

变量使用双大括号表示，用于在模板中显示动态内容：

```html
<h1>{{ page_title }}</h1>
<p>作者：{{ author.name }}</p>
```

当模板引擎遇到变量时，它会计算这个变量，然后将结果插入到输出中。变量可以是简单值，也可以是对象的属性或字典的键。

#### 标签

标签使用`{% %}`语法，用于在模板中执行逻辑操作：

```html
{% if user.is_authenticated %}
    <p>欢迎，{{ user.username }}！</p>
{% else %}
    <p>请登录</p>
{% endif %}

{% for item in items %}
    <li>{{ item.name }}</li>
{% empty %}
    <li>没有项目</li>
{% endfor %}
```

常用的标签包括：
- `if`/`elif`/`else`：条件判断
- `for`/`empty`：循环遍历
- `block`/`extends`：模板继承
- `include`：包含其他模板
- `url`：反向解析URL
- `csrf_token`：CSRF保护

#### 过滤器

过滤器使用管道符号（`|`）表示，用于修改变量的输出：

```html
{{ name|lower }}
{{ text|truncatewords:30 }}
{{ date|date:"Y-m-d" }}
```

常用的过滤器包括：
- `lower`/`upper`：转换为小写/大写
- `truncatewords`：截断文本
- `date`：格式化日期
- `default`：设置默认值
- `safe`：标记内容为安全（不转义HTML）

#### 注释

注释使用`{# #}`语法，用于在模板中添加注释：

```html
{# 这是一个注释 #}
```

多行注释可以使用`{% comment %}`标签：

```html
{% comment %}
这是一个
多行注释
{% endcomment %}
```

### 模板继承

模板继承是Django模板系统的强大特性，允许创建一个基础模板，然后在子模板中扩展和覆盖特定部分：

```html
<!-- base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}默认标题{% endblock %}</title>
    <link rel="stylesheet" href="{% static 'css/style.css' %}">
    {% block extra_css %}{% endblock %}
</head>
<body>
    <header>{% block header %}默认头部{% endblock %}</header>
    <main>{% block content %}{% endblock %}</main>
    <footer>{% block footer %}默认底部{% endblock %}</footer>
    <script src="{% static 'js/main.js' %}"></script>
    {% block extra_js %}{% endblock %}
</body>
</html>
```

```html
<!-- page.html -->
{% extends "base.html" %}

{% block title %}页面标题{% endblock %}

{% block content %}
    <h1>页面内容</h1>
    <p>这是页面的主要内容。</p>
{% endblock %}
```

### 包含模板

`include`标签允许在一个模板中包含另一个模板：

```html
{% include "header.html" %}
<main>页面内容</main>
{% include "footer.html" %}
```

可以传递变量给被包含的模板：

```html
{% include "product_card.html" with product=item only %}
```

`only`选项表示只传递指定的变量，而不传递当前上下文。

## 案例

### 基本模板示例

```html
<!-- templates/blog/article_list.html -->
{% extends "base.html" %}

{% block title %}文章列表{% endblock %}

{% block content %}
    <h1>文章列表</h1>
    
    {% if articles %}
        <ul class="article-list">
            {% for article in articles %}
                <li>
                    <h2><a href="{% url 'article_detail' article.pk %}">{{ article.title }}</a></h2>
                    <p class="meta">作者：{{ article.author.username }} | 发布时间：{{ article.created_at|date:"Y-m-d H:i" }}</p>
                    <p class="summary">{{ article.content|truncatewords:30 }}</p>
                </li>
            {% endfor %}
        </ul>
        
        {% if is_paginated %}
            <div class="pagination">
                {% if page_obj.has_previous %}
                    <a href="?page=1">首页</a>
                    <a href="?page={{ page_obj.previous_page_number }}">上一页</a>
                {% endif %}
                
                {% for num in page_obj.paginator.page_range %}
                    {% if page_obj.number == num %}
                        <span class="current">{{ num }}</span>
                    {% elif num > page_obj.number|add:"-3" and num < page_obj.number|add:"3" %}
                        <a href="?page={{ num }}">{{ num }}</a>
                    {% endif %}
                {% endfor %}
                
                {% if page_obj.has_next %}
                    <a href="?page={{ page_obj.next_page_number }}">下一页</a>
                    <a href="?page={{ page_obj.paginator.num_pages }}">末页</a>
                {% endif %}
            </div>
        {% endif %}
    {% else %}
        <p>暂无文章</p>
    {% endif %}
    
    <p><a href="{% url 'article_create' %}" class="btn btn-primary">发布新文章</a></p>
{% endblock %}
```

### 详情页模板示例

```html
<!-- templates/blog/article_detail.html -->
{% extends "base.html" %}

{% block title %}{{ article.title }}{% endblock %}

{% block content %}
    <article class="article-detail">
        <header>
            <h1>{{ article.title }}</h1>
            <p class="meta">作者：{{ article.author.username }} | 发布时间：{{ article.created_at|date:"Y-m-d H:i" }}</p>
            
            {% if article.category %}
                <p class="category">分类：<a href="{% url 'category_articles' article.category.slug %}">{{ article.category.name }}</a></p>
            {% endif %}
            
            {% if article.tags.all %}
                <p class="tags">
                    标签：
                    {% for tag in article.tags.all %}
                        <a href="{% url 'tag_articles' tag.slug %}" class="tag">{{ tag.name }}</a>{% if not forloop.last %}, {% endif %}
                    {% endfor %}
                </p>
            {% endif %}
        </header>
        
        <div class="content">
            {{ article.content|safe }}
        </div>
        
        <footer>
            {% if user == article.author %}
                <p>
                    <a href="{% url 'article_update' article.pk %}" class="btn btn-secondary">编辑</a>
                    <a href="{% url 'article_delete' article.pk %}" class="btn btn-danger">删除</a>
                </p>
            {% endif %}
            
            <div class="share">
                <h3>分享文章</h3>
                {% include "blog/includes/share_buttons.html" with url=request.build_absolute_uri title=article.title %}
            </div>
            
            {% if article.allow_comments %}
                <div class="comments">
                    <h3>评论 ({{ comments.count }})</h3>
                    
                    {% if comments %}
                        <ul class="comment-list">
                            {% for comment in comments %}
                                <li id="comment-{{ comment.pk }}">
                                    <p class="meta">{{ comment.author_name }} | {{ comment.created_at|date:"Y-m-d H:i" }}</p>
                                    <div class="content">{{ comment.content }}</div>
                                </li>
                            {% endfor %}
                        </ul>
                    {% else %}
                        <p>暂无评论</p>
                    {% endif %}
                    
                    <h4>发表评论</h4>
                    <form method="post" action="{% url 'article_comment' article.pk %}">
                        {% csrf_token %}
                        {{ comment_form.as_p }}
                        <button type="submit" class="btn btn-primary">提交</button>
                    </form>
                </div>
            {% endif %}
        </footer>
    </article>
{% endblock %}
```

### 表单模板示例

```html
<!-- templates/blog/article_form.html -->
{% extends "base.html" %}

{% block title %}{% if form.instance.pk %}编辑文章{% else %}发布新文章{% endif %}{% endblock %}

    <h1>{% if form.instance.pk %}编辑文章{% else %}发布新文章{% endif %}</h1>

            <div class="alert alert-danger">
                {% for error in form.non_field_errors %}

                {% endfor %}
            </div>
        {% endif %}
        
        <div class="form-group">
            {{ form.title.label_tag }}
            {{ form.title }}
            {% if form.title.errors %}
                <div class="error">{{ form.title.errors }}</div>
            {% endif %}
            {% if form.title.help_text %}
                <div class="help-text">{{ form.title.help_text }}</div>
            {% endif %}
        </div>
        
        <div class="form-group">
            {{ form.category.label_tag }}
            {{ form.category }}
            {% if form.category.errors %}
                <div class="error">{{ form.category.errors }}</div>
            {% endif %}
        </div>
        
        <div class="form-group">
            {{ form.content.label_tag }}
            {{ form.content }}
            {% if form.content.errors %}
                <div class="error">{{ form.content.errors }}</div>
            {% endif %}
        </div>
        
        <div class="form-group">
            {{ form.tags.label_tag }}
            {{ form.tags }}
            {% if form.tags.errors %}
                <div class="error">{{ form.tags.errors }}</div>
            {% endif %}
            <div class="help-text">多个标签请用逗号分隔</div>
        </div>
        
        <div class="form-check">
            {{ form.allow_comments }}
            {{ form.allow_comments.label_tag }}
        </div>
        
        <div class="form-actions">
            <button type="submit" class="btn btn-primary">保存</button>
            <a href="{% url 'article_list' %}" class="btn btn-secondary">取消</a>
        </div>
    </form>
{% endblock %}

{% block extra_js %}
    <script src="{% static 'js/markdown-editor.js' %}"></script>
    <script>
        document.addEventListener('DOMContentLoaded', function() {
            initMarkdownEditor('#{{ form.content.id_for_label }}');
        });
    </script>
{% endblock %}
```

## 总结

1. **Django的模板层提供了强大而灵活的模板系统**，支持变量插值、控制流、继承等功能，使开发者能够构建动态的Web页面。

2. **模板语言包含变量、标签、过滤器和注释**，提供了丰富的功能来处理和展示数据，同时保持HTML代码的简洁性。

3. **模板继承是Django模板系统的核心特性**，允许创建一个基础模板，然后在子模板中扩展和覆盖特定部分，提高代码复用性和可维护性。

4. **包含模板机制**允许将常用的UI组件抽取为独立的模板文件，提高代码的模块化和可维护性。

5. **Django模板系统的设计理念是将表现逻辑与业务逻辑分离**，使前端开发者和后端开发者能够更好地协作，提高开发效率。
