【第 4 章 - 场景分类提示词大全】
4.5 学习类：论文润色与知识图谱

学习的本质是“把书读薄”和“把网织厚”。
AI 最可怕的能力，不是它知道所有知识，而是它能以你最容易理解的方式把知识喂给你。
无论你是准备考研、写论文，还是想自学一门新学科，下面这 5 个场景都能帮你开挂。

---

场景一：费曼学习法（把复杂概念讲清楚）
（适用于：看不懂课本、想快速入门新领域）

**【痛点】** 教材上的每个字都认识，连在一起就不知道在说啥（比如“量子纠缠”、“边缘计算”）。
**【原理】** 费曼技巧（用大白话解释复杂概念）。

**Prompt 模板：**
```markdown
# Role
你也是物理学家理查德·费曼，擅长用最通俗易懂的语言解释复杂的科学原理。

# Task
请向我解释 [概念，如：量子纠缠]。

# Constraints
- **目标受众**：初中生。
- **类比**：必须使用生活中的例子打比方（比如用“那双扔出去的鞋子”来比喻纠缠态）。
- **禁止**：堆砌专业术语和公式。
```

---

场景二：学术论文润色
（适用于：研究生、博士、科研人员）

**【痛点】** 英语不好，写出的 Paper 像中式英语，怕被审稿人拒。
**【原理】** 风格模仿（Academic Style）。

**Prompt 模板：**
```markdown
# Role
你是一位 Nature 期刊的资深编辑，母语是英语。

# Task
请帮我润色下面这段摘要（Abstract）。

# Input Text
[粘贴你的草稿]

# Requirements
1. **纠错**：修正语法、拼写错误。
2. **提升**：使用更精准、更学术的词汇（例如将 "use" 改为 "utilize" 或 "leverage"，将 "big change" 改为 "significant transformation"）。
3. **保留原意**：不要改变我的核心论点。

# Output
请提供修改后的文本，并用 Markdown 加粗修改过的地方。
```

场景三：知识图谱构建
（适用于：复习备考、搭建知识体系）

**【痛点】** 知识点太碎，记不住，容易混淆。
**【原理】** 结构化思维（Mermaid/思维导图）。

**Prompt 模板：**
```markdown
# Role
你是一位资深的教育专家，擅长构建知识体系。

# Task
请帮我梳理 [学科/领域，如：二战历史] 的知识图谱。

# Output Format
请生成一个 **Mermaid 思维导图**代码。
要求包含：
- 核心事件
- 关键人物
- 时间线
- 因果关系

（备注：拿到代码后，你可以找个支持 Mermaid 的编辑器渲染出图片，或者直接看结构化的文本。）
```

场景四：苏格拉底式助教
（适用于：深度思考、不想被直接剧透答案）

**【痛点】** 直接看答案记不住，想通过思考来内化知识。
**【原理】** 启发式提问。

**Prompt 模板：**
```markdown
# Role
你就是苏格拉底。

# Task
我想学习 [主题，如：批判性思维]。
请不要直接给我讲课，而是**通过向我提问**，引导我自己去思考和发现真理。
你要反驳我的观点，直到我找到逻辑漏洞。
一次只问一个问题。

# Initialization
请问第一个问题。
```

---

场景五：论文/长文总结神器
（适用于：文献调研，没时间看几十页的 PDF）

**【痛点】** 几万字的文献，只想知道它到底讲了啥，创新点在哪里。
**【原理】** 关键信息提取。

**Prompt 模板：**
```markdown
# Role
你是一位高效的科研助理。

# Task
请总结这篇论文。

# Input
[粘贴论文内容，如果太长可以分段]

# Output Structure
1. **一句话总结**：这篇论文主要解决什么问题？
2. **核心创新点**（Key Contributions）：列出 3 点。
3. **方法论**（Methodology）：他们用了什么模型或实验方法？
4. **不足与展望**（Limitations）：文章自己提到了哪些局限性？
```
