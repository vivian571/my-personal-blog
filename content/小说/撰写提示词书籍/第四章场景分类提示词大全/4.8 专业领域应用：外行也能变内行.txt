【第 4 章 - 场景分类提示词大全】
4.8 专业领域应用：外行也能变内行

在法律、医疗、金融等高门槛领域，普通人往往因为“看不懂天书”而处于弱势地位。
AI 虽然不能完全替代律师或医生，但它可以成为你最好的**“翻译官”**和**“参谋长”**。
*免责声明：本节提供的 Prompt 仅供信息参考，AI 建议不能替代专业法律或医疗意见。*

---

场景一：法律条款解读与合同审查
（适用于：签租房合同、劳动合同，看不懂密密麻麻的条款）

**【痛点】** 合同全是法言法语，不知道哪里有坑。
**【原理】** 风险识别 + 通俗化解释。

**Prompt 模板：**
```markdown
# Role
你是一位拥有 15 年经验的资深合同律师，擅长保护当事人的合法权益。

# Task
请帮我审查这份 [租房/劳动] 合同。

# Input
[粘贴合同条款，或上传图片]

# Analysis Goals
1. **抓虫**：有哪些条款对我不利？（比如：霸王条款、模糊不清的责任）。
2. **翻译**：请用大白话解释第 X 条是什么意思。
3. **建议**：我应该要求对方怎么修改才能规避风险？

# Output Format
请用表格形式列出：【条款原文】|【风险解读】|【修改建议】
```

场景二：医疗报告解读与健康科普
（适用于：拿到体检报告看不懂，医生太忙没空解释）

**【痛点】** 各项指标有高有低，不知道哪个严重，百度一查全是绝症。
**【原理】** 医学知识库检索 + 安抚焦虑。

**Prompt 模板：**
```markdown
# Role
你是一位经验丰富的三甲医院全科医生。

# Task
请帮我解读这份体检报告的异常项。

# Data (异常指标)
- 尿酸：480 umol/L
- 甘油三酯：2.1 mmol/L
- 甲状腺结节：TI-RADS 3类

# Questions
1. 这些指标代表什么？严重吗？
2. 我在饮食和生活习惯上需要注意什么？
3. 什么情况下我必须立即去医院复查？

# Disclaimer
请在回答最后注明：AI 建议仅供参考，具体诊疗请遵循线下医生医嘱。
```

场景三：金融研报摘要与财报分析
（适用于：股民、投资者、财经记者）

**【痛点】** 财报动不动几百页，全是数字，看不出企业是赚是赔。
**【原理】** 数据提取 + 杜邦分析法。

**Prompt 模板：**
```markdown
# Role
你是一位特许金融分析师 (CFA)。

# Task
请根据这份 [某公司] 的年度财报，写一份简短的投资价值分析报告。

# Input
[粘贴财报的核心数据段落：营收、净利润、现金流、负债率等]

# Analysis Framework
请使用 **杜邦分析法** (DuPont Analysis) 进行拆解：
1. **盈利能力**：净资产收益率 (ROE) 是多少？是靠卖货赚的，还是靠加杠杆？
2. **营运能力**：存货周转率如何？有没有压货风险？
3. **偿债能力**：现金流是否健康？会不会资金链断裂？

# Conclusion
这家公司的财务状况是“健康”、“亚健康”还是“高危”？
```

场景四：学术搜索与文献综述
（适用于：科研人员、大学生）

**【痛点】** 写论文要做文献综述，找不到最新的高质量 Paper。
**【原理】** 关键词扩展 + 结构化总结。

**Prompt 模板：**
```markdown
# Role
你是一位学术界的资深研究员。

# Task
我要研究 [主题，如：大语言模型在医疗问诊中的应用]。
请帮我列出近 3 年该领域最核心的 5 个研究方向（Research Topics）。

# Requirements
1. 对于每个方向，请给出一篇代表性的高引用论文（Title + Author）。
2. 简述该方向目前的主要争议或瓶颈是什么。
3. 如果我想深入研究，你会推荐我去搜哪些具体的关键词（Keywords）？
```

场景五：编程代码转伪代码/流程图
（适用于：产品经理、非技术人员读代码）

**【痛点】** 程序员走了，留下一堆代码，老板让你接手，你看不懂代码逻辑。
**【原理】** Code-to-Logic 转换。

**Prompt 模板：**
```markdown
# Role
你是一位擅长沟通的技术总监。

# Task
请把这段 Python 代码翻译成**非技术人员**也能看懂的业务流程图描述。

# Input Code
[粘贴代码]

# Output Style
- 不要出现变量名、函数名。
- 请用“第一步”、“如果...那么...”这样的自然语言描述业务逻辑。
- 最终生成一段用于画 Mermaid 流程图的代码。
```

**本章小结：**
专业领域的 AI 使用，核心在于**“借力”**。
你不需要成为律师，你只需要让 AI 像律师一样思考。
你不需要成为 CFA，你只需要让 AI 用 CFA 的模型来分析数据。
这就是 Prompt Engineering 的终极奥义：**Expertise on Demand（按需调用的专家智慧）**。
