21.2 把代码穿在身上: 3D建模+NFT，做元宇宙里的"LV"

如果你觉得元宇宙地皮太贵，那我们把目光从"住"转向"穿"。

在现实世界，我们愿意花1万块买个LV包包，花5000块买双AJ鞋。
是为了装东西吗？是为了走路快吗？
不，是为了**展示身份(Flexing)**。

在虚拟世界，这个需求不仅存在，而且被放大了100倍。
因为在元宇宙里，你长什么样，完全取决于你的**皮肤(Skin)**和**穿戴(Wearables)**。
没人愿意顶着一个系统默认的"屌丝路人脸"。

于是，一个新的职业诞生了：**数字裁缝 (Digital Tailor)**。
不需要针线，只需要代码和建模软件，你就能成为元宇宙里的LV设计总监。


一、虚拟时尚的暴利: 一双鞋卖了2000万

别以为我在开玩笑。
2021年，一家叫 **RTFKT** 的虚拟球鞋公司，在短短7分钟内卖出了600多双虚拟球鞋，入账310万美元(约2000万人民币)。
后来，这家公司直接被耐克(Nike)收购了。

注意：这些鞋子你是穿不上的，只能给你的虚拟角色穿，或者在你的虚拟展厅里摆着看。
但在Z世代眼里，这就够了。
"我在Instagram上发照片能用，在Decentraland里能穿，这就值钱。"

这就是**"所见即所得"**的数字化消费升级。
传统服装有库存、有物流、有材料成本。
虚拟服装：
- 材料成本 = 0 (只是数据)
- 物流成本 = 0 (链上转账)
- 库 存 = 无限复制
- 利 润 = 100%


二、技术栈: 程序员如何跨界做设计?

你可能会说："我是写代码的，不会搞设计啊。"
其实，元宇宙的穿戴开发，更像是一个**工程问题**，而不是纯艺术问题。

你需要掌握的"三件套"：

1. **Blender (建模)**
   这是开源免费的3D建模软件，极其强大。
   你需要用它捏出鞋子、衣服的形状(Mesh)。
   对于程序员来说，Blender甚至支持 Python 脚本建模。你可以用代码生成几何图形，搞那种很酷的"参数化设计"。

2. **Unity (骨骼绑定)**
   只有模型是不行的，衣服必须跟着人动。
   这叫**蒙皮权重(Rigging & Weighting)**。
   你需要把衣服的顶点，绑定到虚拟角色的骨骼上。
   比如：大腿骨动的时候，裤腿必须跟着弯曲，不能穿模。

3. **glTF 标准 (格式导出)**
   这是3D领域的"JPEG"。
   所有的元宇宙平台(Decentraland, Sandbox, Three.js)都通用这个格式。
   你编写的JSON数据结构，决定了材质、光照和动画。


三、实战: 如何把一件T恤上传到Decentraland?

假设你已经用Blender做了一个简单的T恤模型 `t-shirt.glb`。
接下来的流程全是技术活：

**第一步: 搭建SDK环境**
Decentraland 提供了命令行工具(CLI)。

```bash
npm install -g decentraland
mkdir my-wearable
cd my-wearable
dcl init
```

选择 "Wearable" 项目模板。

**第二步: 配置 metadata (item.json)**
这个文件告诉平台，你这件衣服有什么属性。

```json
{
  "id": "e24d...", 
  "name": "Super Coder T-Shirt",
  "description": "A legendary t-shirt for python masters.",
  "i18n": [{ "code": "en", "text": "Super Coder T-Shirt" }],
  "data": {
    "replaces": [],
    "hides": ["upper_body"], // 穿了这个就隐藏原来的上半身
    "tags": ["cyberpunk", "coder", "python"],
    "category": "upper_body",
    "representations": [
      {
        "bodyShapes": ["urn:decentraland:off-chain:base-avatars:BaseMale"],
        "mainFile": "models/t-shirt_male.glb",
        "contents": ["models/t-shirt_male.glb", "textures/logo.png"],
        "overrideHides": [],
        "overrideReplaces": []
      }
    ]
  }
}
```

**第三步: 本地预览**
运行 `dcl start`。
浏览器会弹出一个本地的元宇宙窗口。
你可以控制一个小人，穿上你设计的这件T恤，走两步，跳一跳。
检查有没有穿模，LOGO会不会变形。

**第四步: 铸造(Mint)并上架**
这里需要支付一笔审核费(通常是 100-500 MANA，约每件几百块钱)。
这是为了防止大家上传垃圾或违规内容。
Decentraland的社区委员会(DAO)会审核你的设计。
一旦通过，这就变成了一个NFT集合(Collection)。
你可以设置：
- 总量：限量100件
- 价格：每件 100 MANA
- 版税：每次二次交易，你抽 5%

一旦上架，你就等着钱包叮叮响吧。


四、进阶玩法: 可编程的衣服

如果只是静态的衣服，那太没技术含量了。
我们要发挥程序员的优势：**做动态的、可交互的衣服。**

**案例1：根据币价变色的卫衣**
你可以写一段代码，连接 Chainlink 预言机。
- 当比特币价格上涨时，卫衣发金光。
- 当比特币价格下跌时，卫衣变绿色(或者红色)。
这种衣服在币圈绝对卖疯。

**案例2：拥有"入场券"功能的鞋子**
把你做的鞋子NFT和你的Discord机器人绑定。
只有钱包里持有这双鞋的人，才能进入你的"VIP频道"。
这就不光是卖鞋，是卖圈子。

**案例3：可成长的宠物背包**
设计一个背在背上的电子宠物。
写一个智能合约，记录你每天在线的时长。
在线越久，这个宠物的等级越高，特效越炫酷。
这会让玩家为了养宠物而疯狂挂机。


五、避坑指南: 设计师的滑铁卢

做虚拟商品，有几个大坑要避开。

**坑一: 版权流氓**
千万不要直接把耐克的LOGO印在衣服上卖。
哪怕是在虚拟世界，品牌方也会告你侵权。
甚至，不要直接抄袭现实中的潮牌设计。
要做原创，或者搞那些现实中做不出来的设计(比如着火的裤子、悬浮的帽子)。
**越反物理，越受欢迎。**

**坑二: 面数爆炸 (Poly Count)**
很多新手做模型，追求细节，一个扣子搞了1万个面。
平台会直接拒绝上架。
因为元宇宙是在网页里跑的，模型面数太高会卡死浏览器。
Decentraland限制每个穿戴模型不超过 1500 个三角形。
**在这里，低模(Low Poly)优化能力比精细建模更重要。**

**坑三: 有价无市**
不要一开始就定天价。
除非你是大V，否则没人会为了一个不知名设计师花几千块。
**策略：免费空投(Free Mint)一部分给社区大V，让他们帮你带货。**
只要有人穿，有了曝光，剩下的自然好卖。


六、总结: 审美红利与技术红利的交汇点

虚拟商品，是**技术(代码) + 艺术(3D) + 经济(NFT)** 的完美结合体。

如果你懂代码，又稍微有一点点审美(或者有一个懂设计的搭档)。
这绝对是一个蓝海市场。

以前，你想做个服装品牌，要搞定面料、工厂、库存、门店，几百万都不够亏的。
现在，你只需要一台电脑。
你的工厂是Blender，你的门店是Decentraland，你的物流是区块链。

就算没卖出去，你也只是浪费了几度电而已。
**这是低成本创业的极致。**
