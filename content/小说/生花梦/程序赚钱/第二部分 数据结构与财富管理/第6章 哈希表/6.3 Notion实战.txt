# 6.3 实战:手把手教你用Notion搭建"个人资源中心"

你好,我是Antigravity。

前面两节,我们讲了信息过滤和人脉管理的理论。

但理论再好,不落地就是空谈。

今天,我要手把手教你,用**Notion**(一个免费的笔记工具),搭建一个完整的"个人资源哈希数据库"。

这个系统包括:
1. **人脉数据库**:管理你的所有联系人,支持多维度检索。
2. **信息收集箱**:过滤和归档高价值信息。
3. **机会追踪器**:记录和跟进各种财富机会。
4. **技能树**:规划你的技能学习路径。

**我保证,如果你花2小时搭建好这个系统,并坚持使用3个月,你的个人效率至少提升30%。**

---

### 一、为什么选择Notion?

市面上的工具很多:Excel、Airtable、飞书多维表格、语雀...

我为什么推荐Notion?

1. **免费**:个人版完全免费,功能够用。
2. **灵活**:既是笔记,又是数据库,还能做看板、日历。
3. **关联**:不同数据库之间可以建立关联关系(这是哈希表的核心)。
4. **美观**:界面简洁,用起来舒服。

**当然,如果你更习惯其他工具,逻辑是一样的,可以自行迁移。**

### 二、搭建"人脉数据库"

#### Step 1:创建数据库

1. 打开Notion,新建一个Page,命名为"个人资源中心"。
2. 在页面里,输入 `/database`,选择"Table - Inline"(内联表格)。
3. 把数据库命名为"人脉网络"。

#### Step 2:设计字段(Properties)

点击表格右上角的"+"号,添加以下字段:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **姓名** | Title | 默认字段,联系人姓名 |
| **标签** | Multi-select | 多选标签,如"Python""创业""深圳" |
| **职业** | Select | 单选,如"产品经理""工程师" |
| **公司** | Text | 文本,当前公司 |
| **认识渠道** | Select | 单选,如"技术大会""朋友介绍" |
| **上次联系** | Date | 日期,最后一次互动时间 |
| **关系温度** | Number | 数字,0-100分 |
| **合作意愿** | Select | 单选,高/中/低 |
| **擅长领域** | Multi-select | 多选,如"用户增长""数据分析" |
| **需求清单** | Text | 文本,对方需要什么 |
| **备注** | Text | 文本,其他信息 |

#### Step 3:录入数据

现在,开始填数据。

**不要一次性填完所有人,那会累死你。**

先填10个核心人脉,然后每周增加5-10个,慢慢积累。

**示例:**

| 姓名 | 标签 | 职业 | 公司 | 上次联系 | 关系温度 | 擅长领域 | 需求清单 |
|------|------|------|------|----------|----------|----------|----------|
| 张三 | Python, 深圳, 创业 | CTO | XX科技 | 2025-12-20 | 85 | 后端开发, 架构设计 | 找前端工程师 |
| 李四 | 短视频, 北京 | 运营总监 | YY传媒 | 2025-11-15 | 60 | 内容运营, 剪辑 | 找投资 |

#### Step 4:建立视图(Views)

Notion的强大之处在于,同一个数据库,可以有多个视图。

点击数据库左上角的"+ Add a view",创建以下视图:

**视图1:按关系温度排序**
- 类型:Table
- 排序:关系温度,降序
- 用途:快速看到哪些核心人脉需要维护

**视图2:按标签筛选**
- 类型:Board(看板)
- 分组:标签
- 用途:快速找到某个领域的人,比如"所有会Python的人"

**视图3:待联系清单**
- 类型:Table
- 筛选:上次联系 < 3个月前,且关系温度 > 50
- 用途:提醒你该联系谁了

### 三、搭建"信息收集箱"

#### Step 1:创建数据库

在"个人资源中心"页面下,再创建一个数据库,命名为"信息收集箱"。

#### Step 2:设计字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **标题** | Title | 信息标题 |
| **来源** | Select | 公众号/知乎/播客/书籍 |
| **分类** | Select | 知识库/待办/灵感/监控 |
| **标签** | Multi-select | Python/创业/理财等 |
| **收集时间** | Date | 自动记录 |
| **优先级** | Select | 高/中/低 |
| **状态** | Select | 未读/已读/已归档 |
| **链接** | URL | 原文链接 |
| **笔记** | Text | 核心要点提炼 |

#### Step 3:建立工作流

**每天早上(信息输入)**
1. 打开Feedly/微信收藏,把值得深读的文章,复制标题和链接到Notion。
2. 快速打上"分类"和"优先级"标签。
3. 状态标记为"未读"。

**每天中午(信息消化)**
1. 打开Notion,筛选"未读 + 优先级高"的条目。
2. 阅读,并在"笔记"栏用自己的话总结核心观点。
3. 状态改为"已读"。

**每周日(信息归档)**
1. 筛选"已读"的条目。
2. 有价值的,转移到"知识库"(另一个数据库,下面会讲)。
3. 没价值的,状态改为"已归档",或直接删除。

### 四、搭建"机会追踪器"

这是最容易被忽视,但最有价值的一个模块。

很多财富机会,都是稍纵即逝的:
- 某个朋友说"我们公司在招人,你有合适的推荐吗?"
- 某个群里有人说"有没有人会做小程序开发?"
- 某个行业报告提到"XX领域未来3年会爆发"

**如果你没有记录和追踪,这些机会就像水一样流走了。**

#### Step 1:创建数据库

新建数据库,命名为"机会追踪器"。

#### Step 2:设计字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **机会描述** | Title | 简短描述 |
| **类型** | Select | 赚钱/人脉/学习/投资 |
| **来源** | Text | 谁告诉你的/在哪看到的 |
| **截止时间** | Date | 机会窗口期 |
| **所需资源** | Text | 需要什么技能/人脉/资金 |
| **行动计划** | Text | 你打算怎么做 |
| **状态** | Select | 待评估/进行中/已完成/已放弃 |
| **结果** | Text | 最终结果如何 |

#### Step 3:实战案例

**案例1:朋友公司招人**
- 机会描述:张三的公司招Python后端,推荐成功有3000元奖金
- 类型:赚钱
- 来源:张三,微信聊天
- 截止时间:2026-01-15
- 所需资源:认识合适的Python工程师
- 行动计划:在人脉数据库里筛选"标签=Python"的人,逐个询问
- 状态:进行中

**案例2:行业趋势**
- 机会描述:AI Agent方向未来3年会爆发
- 类型:学习+投资
- 来源:某VC的行业报告
- 截止时间:2026-12-31
- 所需资源:学习AI Agent开发,关注相关创业公司
- 行动计划:每周学习2小时,关注5个相关项目
- 状态:进行中

### 五、搭建"技能树"

最后一个模块,用来规划你的技能成长路径。

#### Step 1:创建数据库

新建数据库,命名为"技能树"。

#### Step 2:设计字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **技能名称** | Title | 如"Python""短视频剪辑" |
| **分类** | Select | 技术/商业/沟通/创作 |
| **掌握程度** | Select | 入门/熟练/精通/专家 |
| **市场价值** | Number | 1-10分,这个技能值多少钱 |
| **学习优先级** | Select | 高/中/低 |
| **学习资源** | Text | 书籍/课程/项目 |
| **实践项目** | Text | 用这个技能做了什么 |

#### Step 3:建立视图

**视图1:技能矩阵**
- 类型:Board
- X轴:掌握程度
- Y轴:市场价值
- 用途:看到哪些技能是"高价值但未掌握"的,优先学习

**视图2:学习计划**
- 类型:Table
- 筛选:学习优先级=高,且掌握程度<熟练
- 排序:市场价值,降序
- 用途:明确下一步该学什么

### 六、关联魔法:让数据库"活"起来

Notion最强大的功能,是**数据库关联(Relation)**。

比如:
- 在"人脉数据库"里,你可以关联"机会追踪器",记录"这个机会是谁告诉我的"。
- 在"信息收集箱"里,你可以关联"技能树",标记"这篇文章帮我提升了哪个技能"。

**操作方法:**
1. 在"人脉数据库"里,添加一个字段,类型选"Relation"。
2. 选择关联到"机会追踪器"数据库。
3. 这样,每个联系人,都可以链接到他提供的机会。

**这就是哈希表的"链式结构"——不同的数据,通过指针(Relation)连接起来,形成一张网。**

### 七、避坑指南:Notion使用的常见错误

#### 错误1:过度设计

很多人刚开始用Notion,恨不得建100个数据库,每个数据库有50个字段。

结果是:系统太复杂,根本用不起来。

**正确做法:**
- 先从最简单的版本开始,只建3-4个核心字段。
- 用1个月,发现不够用了,再加字段。
- **能用就好,不要追求完美。**

#### 错误2:只建不用

有些人,花3天搭建了超级完美的系统,然后再也没打开过。

**正确做法:**
- 搭建系统的时候,就同步录入数据。
- 每天至少打开一次,养成习惯。
- 设置手机提醒:"每天9点,打开Notion"。

#### 错误3:不做备份

Notion是云端工具,虽然很稳定,但万一账号被封/数据丢失,你就哭了。

**正确做法:**
- 每月导出一次数据(Settings → Export all workspace content)。
- 存到网盘或本地硬盘。

### 本章总结

哈希表的核心思想,是**快速定位**。

在财富管理中,这意味着:
1. **信息过滤**:在海量信息中,快速找到高价值内容。
2. **人脉管理**:在庞大通讯录中,快速定位到你需要的资源。
3. **机会捕捉**:在稍纵即逝的机会中,快速识别和行动。

Notion只是一个工具,真正重要的是**系统化思维**。

**记住:工具是死的,人是活的。再好的系统,不用也是零。**
