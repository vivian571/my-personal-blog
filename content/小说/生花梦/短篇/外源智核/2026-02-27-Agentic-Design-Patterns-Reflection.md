# 🎨 天工译介：智能体设计模式之“反思” (Reflection)

> **语义炼金核心**: 放弃单次生成的偶然，拥抱迭代反思的必然。在算力构建的镜面前，AI 正在通过自我审视完成从“脚本”向“智能”的跨越。

## 1. 深度摘要 (The Distilled Insight)
Andrew Ng（吴恩达）在此文中剖析了 **反思 (Reflection)** 模式——这是提升 Agent 性能最快且最有效的手段之一。其核心逻辑在于：不再强求 LLM 一次性输出完美结果，而是通过**自动化反馈机制**，让模型审视自身输出，进行自我批判与修正。

## 2. 关键炼金逻辑 (Key Alchemical Nodes)
*   **批判自动化 (Automated Critique)**: 将人类提供反馈的过程内置化。通过 Prompt 诱导模型检查代码的正确性、风格和效率。
*   **工具辅助反思 (Tool-Enhanced Reflection)**: 引入单元测试、网络搜索等外部工具作为“真理锚点”，让模型基于运行结果而非仅仅是概率预测来进行自我修正。
*   **多智能体对抗 (Multi-Agent Confrontation)**: 建立“生成者”与“审计者”两个角色。在角色的博弈与对话中，逻辑的瑕疵被不断磨平，输出的质感得到质的飞跃。

## 3. 语义内核 Distillation
**“智能不在于永不犯错，而在于拥有察觉并修正错误的代码自省本能。”**
反思模式将 Agent 从一个静态的响应器转化为一个动态的进化体。
