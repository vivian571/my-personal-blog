10.2 方法定义:会说会写会合作,你的Person类才值钱

你好,我是Antigravity。

上一节,我们定义了Person类的属性:技能、口碑、影响力。

但光有属性还不够,你还需要有方法(Methods)。

在编程中,方法就是对象能做的事情。比如汽车能drive(),人能walk()。

今天,我们要给你的Person类,定义三个最重要的方法:演讲()、写作()、合作()。

为什么是这三个?

因为在知识经济时代,会说、会写、会合作的人,才能把知识变现。


一、为什么方法比属性更重要?

很多人,技能很强(属性高),但不会表达(方法缺失)。

结果是:
- 技术很牛,但没人知道。
- 能力很强,但赚不到钱。

举例:

小王,Python技术9分,但不会演讲,不会写作,不会合作。
- 他只能做技术外包,时薪200元。

小李,Python技术7分,但会演讲,会写作,会合作。
- 他做技术培训,时薪1000元。
- 他写技术书,版税收入10万/年。
- 他和别人合作做产品,分成收入50万/年。

技能差距只有2分,但收入差距是10倍。

这就是方法的力量。


二、方法1:演讲() - 把知识变成影响力

演讲,不是站在台上讲话那么简单。

演讲的本质,是用语言影响别人。

为什么演讲重要?

1. 建立权威:能上台讲,说明你是专家。
2. 扩大影响力:一次演讲,能触达几百人。
3. 获得机会:很多合作,都是从演讲开始的。

如何提升演讲能力?

第一步:克服恐惧

很多人,一上台就紧张,脑子一片空白。

怎么办?

1. 从小场景开始:先在团队会议上发言,再到部门会议,再到公司大会。
2. 多练习:每次演讲前,至少练习3遍。
3. 录视频:用手机录下自己的演讲,看看哪里可以改进。

第二步:设计内容框架

一个好的演讲,必须有清晰的结构。

我推荐的框架:

1. 开场(1分钟):抛出一个问题或故事,吸引注意力。
   - 错误开场:"大家好,我是XX,今天我要讲XX。"(无聊)
   - 正确开场:"你有没有遇到过这样的情况:写了一天代码,结果发现需求理解错了?"(共鸣)

2. 主题(2分钟):说明今天要讲什么,为什么重要。
   - "今天我要分享,如何用需求评审,避免90%的返工。"

3. 案例(10分钟):用真实案例,讲解你的方法。
   - "去年我们团队有个项目,因为需求不清晰,返工了3次。后来我们引入了需求评审流程,问题解决了。"

4. 互动(2分钟):提问,让听众参与。
   - "大家有没有遇到过类似的问题?有的话举个手。"

5. 收尾(1分钟):总结要点,给出行动建议。
   - "今天的核心观点是:需求评审很重要。建议大家下周就在团队里试试。"

第三步:练习表达技巧

1. 语速:不要太快,也不要太慢。正常语速是每分钟150-180字。
2. 停顿:讲完一个观点,停顿2秒,让听众消化。
3. 眼神交流:不要只看PPT,要看听众。
4. 肢体语言:适当用手势,增强表达力。

实战案例:我的第一次技术分享

2020年,我第一次在公司做技术分享,主题是"Python性能优化"。

准备过程:
- 花了3天准备内容,写了30页PPT。
- 练习了5遍,录了视频,发现自己语速太快。
- 调整后,再练习3遍。

现场表现:
- 开场:我讲了一个故事:"去年双11,我们的系统崩了,因为性能优化没做好。"(大家都笑了,气氛活跃了)
- 主题:我分享了5个性能优化技巧。
- 互动:我问:"大家有没有遇到过性能问题?"(很多人举手)
- 收尾:我总结了核心观点,并分享了一个性能优化工具。

结果:
- 分享很成功,老板表扬了我。
- 有3个同事,会后来找我请教。
- 我的技术影响力,在公司里提升了。

避坑指南:

1. 不要照着PPT念:听众会睡着。
2. 不要讲太多细节:抓住核心观点就行。
3. 不要超时:说好20分钟,就不要讲30分钟。
4. 不要忽视提问:听众提问,是最好的互动机会。


三、方法2:写作() - 把知识变成资产

写作,是最容易被低估的技能。

很多人觉得:"我又不是作家,为什么要学写作?"

但真相是:在互联网时代,会写作的人,赚钱更容易。

为什么写作重要?

1. 可复制:一篇文章,可以被无数人看到。
2. 可积累:文章是资产,会持续带来流量和收入。
3. 可变现:写作可以直接赚钱(公众号、知识星球、付费专栏)。

如何提升写作能力?

第一步:选题

好的选题,成功了一半。

选题的三个标准:
1. 有需求:别人想看。
2. 有差异化:别人没写过,或者你能写得更好。
3. 有价值:看完后,读者能学到东西。

举例:

错误选题:"Python入门教程"(太泛,竞争激烈)
正确选题:"Python自动化办公:3个脚本,帮你每天省2小时"(具体,有价值)

第二步:大纲

不要上来就写,先列大纲。

我的大纲模板:

1. 开头:抛出问题,引起共鸣。
2. 背景:为什么这个问题重要?
3. 方法:如何解决这个问题?(分3-5个小点)
4. 案例:真实案例,证明方法有效。
5. 总结:核心观点,行动建议。

第三步:写作

写作的核心原则:简单、清晰、有用。

1. 用短句:一句话不超过30字。
2. 用口语:像和朋友聊天一样写。
3. 用例子:抽象的概念,用例子解释。
4. 用小标题:每300字,加一个小标题,方便阅读。

第四步:修改

好文章,都是改出来的。

我的修改流程:
1. 第一遍:检查逻辑,是否清晰?
2. 第二遍:检查语言,是否简洁?
3. 第三遍:检查错别字,是否有低级错误?

实战案例:我的公众号写作经历

2020年,我开始写公众号,主题是"Python实战"。

第一篇文章:
- 标题:"Python爬虫入门"
- 阅读量:50(惨淡)

我分析了原因:
- 标题太泛,没有吸引力。
- 内容太理论,没有实战案例。

调整后:
- 标题:"用Python爬取豆瓣电影Top250,10分钟搞定"
- 内容:手把手教学,有代码,有截图。
- 阅读量:2000(成功!)

关键:
- 标题要具体,要有数字。
- 内容要实战,要有案例。
- 排版要清晰,要有小标题。

避坑指南:

1. 不要追求完美:先发布,再优化。
2. 不要写太长:3000字以内最好,超过5000字,读者会累。
3. 不要只写理论:一定要有案例,有代码。
4. 不要忽视标题:标题决定了80%的阅读量。


四、方法3:合作() - 把资源变成收入

合作,是最被忽视的赚钱方式。

很多人,习惯单打独斗,结果:
- 效率低,什么都要自己做。
- 天花板低,一个人能力有限。

但会合作的人,能整合资源,放大收益。

为什么合作重要?

1. 优势互补:你擅长技术,别人擅长营销,合作能1+1>2。
2. 分担风险:创业风险大,合作能分担。
3. 扩大规模:一个人做不了的事,团队能做。

如何找到合作伙伴?

第一步:明确你的需求

你需要什么样的合作伙伴?

举例:
- 你擅长技术,需要一个擅长营销的合伙人。
- 你擅长产品,需要一个擅长运营的合伙人。

第二步:在哪里找?

1. 行业社群:加入技术社群、创业社群。
2. 线下活动:参加技术大会、创业沙龙。
3. 朋友介绍:让朋友帮忙介绍。

第三步:如何谈合作?

1. 先建立信任:不要一上来就谈钱,先聊聊彼此的想法。
2. 明确分工:谁负责什么,要清晰。
3. 谈好分成:收益怎么分,要提前说清楚。
4. 签合同:口头约定不靠谱,一定要签合同。

合作的三种模式:

模式1:技术咨询
- 你提供技术服务,客户付费。
- 定价:按小时收费,或按项目收费。
- 适合:技术能力强,但不想创业的人。

模式2:联合开发
- 你负责技术,合伙人负责营销,收益分成。
- 分成比例:通常是50:50,或60:40。
- 适合:想创业,但缺少营销能力的人。

模式3:技术入股
- 你用技术入股,不拿工资,拿股份。
- 股份比例:根据贡献,通常是10%-30%。
- 适合:看好项目,愿意承担风险的人。

实战案例:我的第一次商业合作

2021年,我和一个朋友合作,做了一个SaaS产品。

分工:
- 我负责技术:产品开发、服务器部署。
- 他负责营销:内容运营、用户增长。

分成:
- 收益五五分。
- 我拿50%,他拿50%。

结果:
- 第一年:收入10万,我分5万。
- 第二年:收入30万,我分15万。
- 第三年:收入80万,我分40万。

关键:
- 分工明确,各司其职。
- 分成公平,没有矛盾。
- 定期沟通,及时调整策略。

避坑指南:

1. 不要和不熟的人合作:至少要认识半年以上。
2. 不要口头约定:一定要签合同,写清楚分工和分成。
3. 不要忽视沟通:每周至少开一次会,同步进度。
4. 不要贪心:分成要公平,不要想着占便宜。
5. 不要拖延:说好的事情,要按时完成,不要让合伙人失望。


五、如何把三个方法组合起来?

演讲、写作、合作,不是孤立的,而是相互促进的。

举例:

第一步:写作
- 你写了一篇技术文章,发到公众号。
- 阅读量5000,有100个人加你微信。

第二步:演讲
- 你在技术大会上,做了一次分享。
- 现场200人,会后有10个人加你微信。

第三步:合作
- 你的文章和演讲,吸引了一个创业者。
- 他找你合作,一起做产品。
- 你负责技术,他负责营销,收益分成。

这就是一个完整的闭环:
- 写作→建立影响力
- 演讲→扩大影响力
- 合作→变现影响力


六、工具:Python实现个人方法评估系统

最后,分享一个工具,评估你的方法能力。

class PersonMethods:
    def init(self, name):
        self.name = name
        self.speaking_score = 0
        self.writing_score = 0
        self.collaboration_score = 0
    
    def evaluate_speaking(self, metrics):
        计算演讲能力分数
        metrics = {
            'presentations_count': 10,  演讲次数
            'avg_audience': 50,  平均听众数
            'feedback_score': 4.5  反馈评分(1-5)
        }
        
        count_score = min(50, metrics['presentations_count'] * 5)
        audience_score = min(30, metrics['avg_audience'] / 5)
        feedback_score = (metrics['feedback_score'] / 5) * 20
        
        self.speaking_score = count_score + audience_score + feedback_score
        return self.speaking_score
    
    def evaluate_writing(self, metrics):
        计算写作能力分数
        metrics = {
            'articles_count': 50,  文章数量
            'total_views': 100000,  总阅读量
            'avg_engagement': 0.05  平均互动率
        }
        
        count_score = min(30, metrics['articles_count'] / 2)
        views_score = min(40, metrics['total_views'] / 2500)
        engagement_score = metrics['avg_engagement'] * 600
        
        self.writing_score = count_score + views_score + engagement_score
        return self.writing_score
    
    def evaluate_collaboration(self, metrics):
        计算合作能力分数
        metrics = {
            'projects_count': 5,  合作项目数
            'success_rate': 0.8,  成功率
            'total_revenue': 50  总收入(万元)
        }
        
        count_score = min(30, metrics['projects_count'] * 6)
        success_score = metrics['success_rate'] * 30
        revenue_score = min(40, metrics['total_revenue'] / 2)
        
        self.collaboration_score = count_score + success_score + revenue_score
        return self.collaboration_score
    
    def generate_report(self):
        生成报告
        report = f"\n=== {self.name}的方法能力评估报告 ===\n"
        report += f"\n演讲能力: {self.speaking_score:.0f}/100\n"
        report += f"写作能力: {self.writing_score:.0f}/100\n"
        report += f"合作能力: {self.collaboration_score:.0f}/100\n"
        
        avg_score = (self.speaking_score + self.writing_score + self.collaboration_score) / 3
        report += f"\n综合评分: {avg_score:.0f}/100\n"
        
        if avg_score >= 80:
            level = "大师级"
        elif avg_score >= 60:
            level = "高级"
        elif avg_score >= 40:
            level = "中级"
        else:
            level = "初级"
        
        report += f"当前等级: {level}\n"
        return report

使用示例
me = PersonMethods("Antigravity")

演讲能力评估
me.evaluate_speaking({
    'presentations_count': 10,
    'avg_audience': 50,
    'feedback_score': 4.5
})

写作能力评估
me.evaluate_writing({
    'articles_count': 50,
    'total_views': 100000,
    'avg_engagement': 0.05
})

合作能力评估
me.evaluate_collaboration({
    'projects_count': 5,
    'success_rate': 0.8,
    'total_revenue': 50
})

生成报告
print(me.generate_report())


本节小结

Person类的三个核心方法:
1. 演讲():用语言影响别人,建立权威。
2. 写作():用文字积累资产,持续变现。
3. 合作():用资源整合能力,放大收益。

记住:属性决定你的潜力,方法决定你的收入。
