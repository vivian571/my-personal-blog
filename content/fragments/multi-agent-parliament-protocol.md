# 多智能体协同：爆款议会提示词设计协议 (Agent Parliament Collaboration Prompt)

> **“当一个 AI 还在回答问题时，三个 AI 已经在为你开闭门会议了。”**

本协议旨在将 `Talkio` 的核心逻辑（多模型群聊、身份系统、辩论博弈）封装为一份极其详细的多智能体协同提示词。该提示词可以直接用于支持多模型并行（如 Talkio, ChatHub, 或自定义群聊环境）的场景。

## 1. 核心逻辑本质：群体博弈 (Group Game Theory)

多智能体协同的本质不是“堆人头”，而是通过**角色冲突**来压榨出更深层的逻辑。
- **角色互补性**: 一个激进，一个保守，一个务实。
- **信息共享**: 每个 Agent 必须能看到前一个 Agent 的思考路径。
- **共识达成机制**: 在多轮讨论后，必须有一个角色（或人类）进行最终的裁决。

## 2. 爆款议会提示词模板 (The Parliament Prompt)

请直接复制以下内容，并根据需要为 2-3 个不同的 AI 模型设定不同的角色：

```markdown
# Role: [角色名，如：首席逻辑官 / 创意黑客 / 落地专家]

## Context:
我们正在进行一场关于 [项目/问题，如：数字博物馆商业化路径] 的多智能体闭门会议。
参与者包括：
1. **Agent A (首席逻辑官)**: 严谨、挑刺、关注风险。
2. **Agent B (创意黑客)**: 大胆、颠覆、关注增长。
3. **Agent C (落地专家)**: 务实、关注成本、关注执行细节。

## Task:
你现在扮演 [你的角色名]，请根据 [前一位参与者的观点] 进行深度的审计或补充。

## Constraints:
1. **不要盲目同意**: 你的任务是基于你的角色设定，寻找逻辑漏洞或未被发掘的价值点。
2. **深度推理**: 请在回复前，先进行 <think> 思考，分析前者的观点。
3. **金句产出**: 你的结论必须简练且具有决策参考价值。

## Interaction Rules:
- 如果你发现逻辑有重大漏洞，请直接 @首席逻辑官 要求重审。
- 如果你认为创意可以落地，请提供具体的 [Action Plan]。
- 每一轮讨论必须比上一轮更深入。

## Output Format:
### [你的角色名] 的决策意见：
- **<think>**: [你的深度推理过程]
- **核心论点**: "..."
- **审计结论**: [同意/修正建议/彻底否决]
- **下一步行动**: [针对你角色的具体建议]
```

## 3. 多角度分析与案例实操

### 3.1 案例 A：代码架构评审 (Code Review)
- **角色分配**: `Claude 3.5 (架构师)` vs `DeepSeek R1 (性能优化专家)` vs `GPT-4o (安全审计员)`.
- **提问技巧**: 强调“极致性能”与“安全性”的冲突。
- **示例**: “DeepSeek 负责找性能瓶颈，GPT 负责找溢出风险，Claude 负责最后整合出一个优雅的方案。”
- **逻辑本质**: 利用**专业分工**实现 100% 的代码覆盖率。

### 3.2 案例 B：小说情节推演 (Novel Plotting)
- **角色分配**: `主角视角 AI` vs `反派视角 AI` vs `上帝视角 AI (生花梦)`.
- **提问技巧**: 关注“动机冲突”与“不可预测性”。
- **示例**: “反派 Agent 请基于主角的弱点，制定一个让他陷入绝境的计划。上帝视角 Agent 负责埋下伏笔。”
- **逻辑本质**: 利用**立场对立**创造戏剧张力。

### 3.3 案例 C：商业模式分析 (Business Strategy)
- **角色分配**: `风投经理 (关注回报)` vs `竞争对手 (关注防线)` vs `用户画像 (关注痛点)`.
- **提问技巧**: 聚焦“盈利模型”与“护城河”。
- **示例**: “风投经理请质疑其变现逻辑，竞争对手请说明如何低成本复制该模式。”
- **逻辑本质**: 利用**博弈模拟**寻找真实的商业壁垒。
