# 数字博物馆 - 下一步行动计划 (PLAN.md)

*创建时间: 2026-02-09*
*对应目标: 优化博客体验，整合技能展示，增强互动性*

## 1. 现状诊断
- **运行状态**: 服务正常 (Gateway + Blog)。
- **痛点**:
    - 缺乏内容检索机制 (搜索/标签)。
    - 存在潜在安全隐患 (API 证书验证被禁用)。
    - 交互性不足 (无评论)。
- **资产**: 拥有丰富的技能文档 (`content/skills/*.md`) 可转化为优质内容。

## 2. 阶段性目标

### 阶段一：基础加固 (本周)
- [ ] **安全修复**: 移除 `client.ts` 中的 `rejectUnauthorized = false`，配置正确的证书或环境变量。
- [ ] **运维清理**: 编写脚本清理 `/app/config/agents/*/sessions/` 下的过期会话日志。
- [ ] **性能分析**: 使用 LightHouse 分析首页性能，针对性优化图片和脚本加载。

### 阶段二：功能增强 (下周)
- [ ] **搜索功能**: 实现基于 Fuse.js 或简单的服务端关键词匹配的全文搜索。
- [ ] **标签系统**: 解析 Markdown frontmatter 中的 `tags` 字段，生成标签云和分类页。
- [ ] **技能展示页**: 专门开发 `/skills` 路由，美观地展示 `content/skills/` 下的所有技能卡片。

### 阶段三：互动与扩展 (未来)
- [ ] **评论系统**: 调研 Giscus (GitHub Discussions) 或自建简易评论区。
- [ ] **RSS 订阅**: 生成 `feed.xml`，方便读者订阅更新。
- [ ] **公众号联动**: 利用 `wechat-article-persona` 将博客精选内容转换为公众号推文风格。

## 3. 技能包整合策略
利用现有的 `content/skills/` 文件，将其作为博客的一个独特板块 "AI Skills"。
- **自动化**: 编写脚本读取该目录下所有 `.md` 文件。
- **展示**: 提取 `name` 和 `description`，生成可视化的技能卡片墙。

请指示优先启动哪一项任务？建议从 **阶段一：基础加固** 开始，或直接进行 **阶段二：技能展示页** 的开发以丰富内容。
